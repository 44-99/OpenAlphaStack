不是“智能炒股神器”

研究可以灵活，执行必须确定

我们不让一个模型同时扮演分析师、风控、交易所和审计员。每一层只做自己擅长的事，失败、过期和缺失数据都必须被明确暴露。

研

研究方法放进 Skills

市场情绪、板块轮动、候选筛选和个股分析成为可复用的方法，而不是散落在聊天记录里的临时提示词。

默认由一个 Codex 任务按需组合，不制造没有必要的多智能体戏剧。

界

实时能力放进 MCP

行情、技术面、基本面、新闻、风险计算和基准回测通过稳定参数与版本化响应暴露。

每个响应保留来源、截至时间、新鲜度和示例数据标记。

规

硬规则留给 Python

T+1、费用、仓位上限、幂等、状态迁移和账本提交由确定性代码执行，不由模型自行解释。

公开能力只有研究、回测与模拟交易，不连接真实券商。

四层清晰边界

从研究意图到可复核证据

智能体负责提出和解释，工具负责读取，规则负责执行，账本负责留下证据。

第一层 · 对话与调度

Codex 任务

理解研究意图、组合方法、引用证据并生成可审阅结论。

第二层 · 研究方法

领域 Skills

封装市场分析、候选筛选、个股研究与底仓日内约束。

第三层 · 类型化能力

MCP 工具

第四层 · 规则与证据

Python 与 SQLite

边界比口号更重要

它不会替你承诺收益

回测和模拟结果只是研究证据。OpenAlphaStack 的价值是让假设、数据、规则、状态和结果更容易被检查，而不是把不确定性包装成确定答案。

真实券商下单 任意命令执行 收益保证 缺失数据自动编造

常见问题

开始之前，先把边界说清楚

这是自动实盘交易软件吗？

不是。公开模式只有研究、回测与模拟交易，不提供真实券商连接和下单工具。

为什么默认不启动多个子智能体？

大多数研究任务由一个 Codex 任务组合领域 Skills 已经足够。只有用户明确要求独立并行工作时才考虑子智能体。

公开网站会暴露我的本地数据吗？

不会。本站是静态页面；本地观察面板默认绑定 127.0.0.1。独立公网 MCP 也只保留无状态只读工具。

真实行情不可用时还能体验吗？

可以。仓库提供明确标记的合成离线数据，用于验证安装、协议和研究工作流，不作为投资数据。
