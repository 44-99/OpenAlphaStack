Public MCP checks

  1. Open the service /health endpoint and confirm an OK response.
  2. Connect the client to the HTTPS /mcp endpoint, not this static website.
  3. Call get_public_capabilities or read_demo_dataset before diagnosing an upstream market-data failure.
  4. Include the time, tool name, input category, and stable error code in a report. Never include credentials.

Security issues

Do not disclose vulnerabilities, leaked credentials, or access to non-public state in an issue or discussion. Follow the repository security policy.

Support boundary

Community maintenance does not provide investment decisions, return promises, managed trading, or emergency trading support. The public service does not connect to a live broker.
