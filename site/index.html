WHY OPENALPHASTACK

减少 Agent 戏剧，增加可验证边界

它不是自动盈利黑盒，而是一套可安装、可检查、可替换的 A 股研究基础设施。

01

领域 Skills

市场分析、选股、个股分析和底仓 T0 按可复用能力组织，由一个 Codex 任务按需组合。

02

有类型的本地 MCP

行情、指标、新闻、回测和模拟计划通过稳定 schema 暴露，不提供 Shell 或真实下单工具。

03

确定性 Python 引擎

T+1、费用、幂等、状态迁移和账本由代码执行；置信度与风险文字只做非阻断审计信息。

THREE CLEAR STAGES

研究、执行、评估，仅此三层

  1. ResearchCodex 组合 Skills 与 MCP 数据，生成有来源的研究结论。
  2. Execution一次 publish_paper_plan 进入确定性硬约束与模拟执行。
  3. EvaluationSQLite、追加式账本和 Dashboard 留下可复核证据。

LOCAL QUICK START

在本地跑起来

需要 Python 3.10+、Node.js 20+ 与 Codex Desktop。

git clone https://github.com/44-99/OpenAlphaStack.git
cd OpenAlphaStack
pip install -e ".[all]"
npm install
npm run dashboard:build
openalphastack doctor
openalphastack app start

打开 http://127.0.0.1:8800/dashboard，再从 Codex 调用 $market-analyzer。网络或行情不可用时可先运行明确标记的离线 Demo。

FAQ

先把边界说清楚

这是自动实盘交易软件吗？

不是。公开模式只有研究、回测与模拟交易，MCP 不提供真实券商下单工具。

需要默认启动多个子 Agent 吗？

不需要。默认由一个 Codex 任务组合领域 Skills；只有用户明确要求独立并行工作时才考虑子 Agent。

静态官网会暴露我的 Dashboard 吗？

不会。本站只有静态 HTML、CSS 与演示素材。本地 Dashboard 仍绑定 127.0.0.1，不能在缺少认证、TLS 和网络策略时公开。

项目是否保证盈利？

不保证。回测和模拟结果只是研究证据，不是收益承诺；亏损和失败样本同样应该被记录与复盘。
